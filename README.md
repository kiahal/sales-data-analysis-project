
#  End-to-End Data Analytics Project

Este es un proyecto integral de análisis de datos que abarca desde el modelado conceptual hasta la visualización avanzada y el storytelling. El objetivo principal fue transformar datos transaccionales crudos en una herramienta de toma de decisiones estratégica para optimizar la rentabilidad.

---

##  Metodología y Herramientas

### 1. Modelado de Datos (Arquitectura)
* **Diseño de DER:** Utilicé **Draw.io** para definir la lógica de negocio mediante un diagrama Entidad-Relación, asegurando la integridad referencial antes de la implementación.
* **Arquitectura:** Implementación de un **Modelo en Estrella (Star Schema)** para optimizar el rendimiento de las consultas y la claridad del modelo de datos.

### 2. Gestión de Datos y SQL Server
* **ETL (Extract, Transform, Load):** Carga y limpieza de registros en **Excel**, aplicando validaciones de datos para asegurar la consistencia.
* **Consultas Avanzadas:** Migración de datos a **SQL Server**, donde desarrollé consultas complejas para validación de inventarios, segmentación de ventas y limpieza profunda de registros de manera eficiente.

### 3. Business Intelligence & DAX (Power BI)
* **Lógica de Negocio (DAX):** Creación de medidas y columnas calculadas mediante lenguaje **DAX** para generar **KPIs dinámicos** (Margen de Contribución, Ticket Promedio, Crecimiento Interanual, etc.).
* **Análisis de KPIs:** Implementación de indicadores clave de rendimiento enfocados en analizar la rentabilidad por producto y el comportamiento de compra de los clientes.
* **UI/UX Design:** Diseño de una interfaz personalizada en **Canva**, priorizando la experiencia de usuario y la navegación intuitiva entre reportes (Ventas, Clientes y Rentabilidad).

### 4. Data Storytelling
* **Comunicación de Insights:** Elaboración de un video explicativo donde detallo el flujo de trabajo técnico y los hallazgos más relevantes del negocio, demostrando la capacidad de traducir datos técnicos en valor estratégico.

## 📂 Estructura del Repositorio

Para facilitar la navegación por el proyecto, el contenido se organiza de la siguiente manera:

* **`/data`**: Contiene el dataset principal en Excel (`Dataset_Ventas_Proyecto.xlsx`) con los registros transaccionales utilizados.
* **`/docs`**: Incluye el **Diagrama Entidad-Relación (DER)** diseñado en Draw.io, tanto en formato editable como en imagen para visualización rápida.
* **`/reports`**: Aquí se encuentra el archivo de **Power BI** (`.pbix`) con el dashboard interactivo y el análisis visual.
* **`/scripts`**: Contiene los archivos **SQL** con las consultas de transformación y análisis de datos.-
